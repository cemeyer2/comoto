Release Notes:



5/3/12:
  • Added optional fisheye focus to viz
  • Removed viz popups in favor of Bootstrap popovers
  • Fixed viz layout to properly layout only data that will be shown
  • Fixed style inconsistancies resulting from including Bootstrap
4/17/12:
  • Viz JavaScripts now self hosted to eliminate insecure warnings on webkit browsers.
  • All JavaScript files are now minified to reduce loading times
  • All JavaScript libraries have been updated to the latest compatible versions.
4/16/12:
  • Comoto 1.5 released!
  • Dynamic visualization is now live. In order for an analysis to appear in the visualization menu, it needs to be pruned to a specific semester. This can be done in the textual view of the analysis under the filtering options. Links to launch the viz are shown on the list analysis page as well as the textual analysis report page only for analyses that have been pruned to a specific semester.
  • JavaScript includes have been fixed to eliminate SSL errors shown in some browsers.
2/10/11:
  • Non-FERPA protected student data is now displayed on each student's history page.
2/8/11:
  • Added new getFileSets API method.
2/1/11:
  • Users can now export, optionally anonymized, moss output files.
1/12/11:
  • Added new getStudents API method.
  • API method getMossAnalysis now returns if it was pruned and the identifier of the pruned offering if it exists and -1 otherwise.
12/21/10:
  • Major database schema changes that required resetting all data and starting from scratch. All future schema changes should be able to be migrated smoothly.
  • Database backend now uses faster engine.
  • Performance improvements when importing sets of files with many students new to the system.
12/10/10:
  • Performance enhancements for roster view when viewing large course rosters
  • API method getFileSet now has a different signature. Setting the full_submission_info parameter to True will return full submission structs for each submission in the file set, including basic student information structs if the submission is a student submission.
  • API method getSubmission now has an optional second parameter that when set to True will return a full student struct for the student that corresponds to that submission if the submission is a student submission.
  • There is now an integrated tool to test the API on the API documentation page.
12/07/10:
  • University class rosters are now integrated into CoMoTo where available. To use this feature, click on one of your courses on the home page and then click on view roster next to the offering of the course for which you want to view the course roster.
  • API methods:
    • getCourse
    • getCourses
    • getFileSet
    • getOffering
    • getStudent
    • getStudentByNetid
    all now return their UIUC AD LDAP dn (distinguished name) for their respective objects if they have been linked.
  • API methods:
    • getCourse
    • getCourses
    • getFileSet
    • getOffering
    all now return their their UIUC AD LDAP information for the offerings returned in the offering_info field and identifiers for each student in their rosters if the optional boolean parameter in each method is set to True.
12/06/10:
  • API stability enhancements. All API functions now pass the test suite.
  • Users can now download source files from the view file page
  • Users can now download submissions from the list file set page
  • Users can now download entire file sets from the list file sets page
11/30/10:
  • Analysis can now be filtered to limit the number of matches that a given student will appear in, which in some cases helps to reduce clutter
  • Static resources are now cached to reduce page load times
  • API Enhancements:
    • getMossAnalysis now includes the ability to return categorized matches as seen on the view analysis page as well as the same filtering abilities as on the view analysis page
    • getStudent now includes the ability to filter the returned list of historical matches
    • getStudentByNetid now includes the ability to filter the returned list of historical matches
11/29/10:
  • Documented the XML-RPC API. Documentation is available here.
11/9/10:
  • Various UI enhancements, especially on view analysis pages
  • Increased the maximum value for moss -m parameter
  • Fixed bug that prevented matches from showing up from a student if that student appeared in multiple semesters in the same analysis run
11/5/10:
  • Fixed bug that sometimes prevented status from appearing when importing files or running analysis
11/4/10:
  • added a variety of new methods to API to allow for accessing of database objects by id
  • static files are now gzip compressed if supported by client browser
  • partners can now be parsed and displayed in analysis results
  • enhanced analysis view ui to include floating links to sections of analysis results
10/27/10:
  • added getStudentByNetid API method call
  • a few minor UI bug fixes
10/26/10:
  • API re-worked to raise XML-RPC faults in most error conditions or security check failures
  • getStudent API method now takes an optional boolean parameter to indicate if the requested student's history should also be returned (defaults to False)
  • API url simplified, is now just /api rather than /api/api
  • Some JavaScript libraries are now loaded from Google, reducing page load times
  • View file page enhanced to show additional data
  • E-Mailing of students from CoMoTo is now all done via AJAX
  • Range of maximum moss matches to calculate when running analysis has been reduced from 5000 to 2000
  • Static graph generation now incorporated into view analysis page
10/15/10:
  • Analysis can now be deleted from the system
  • All delete/remove tasks now require user confirmation
  • Fixed bug preventing files with non-ASCII characters from being imported properly
  • Display of analysis can now be filtered to show matches only scoring above a given threshold
  • UI tweaks on analysis display to indicate when no matches are available
  • Added UI enhancement to filter files page to prevent user causing application error
  • Users can now directly email their students that appear in report results directly from CoMoTo
10/12/10:
  • Student history data is now available. Clicking on the history links will allow you to search for students to display their history in CoMoTo. Clicking on a student's netid anywhere in CoMoTo will also display their history in CoMoTo.
  • Fixed bug that prevented importing base files and solution files via svn
  • Solution and base file offerings are automatically created for new courses when they are created
  • Analysis can be pruned down to one semester such that only matches that have a student in the semester that you prune to appear in the analysis results
10/11/10:
  • Bug fixed in dynamic moss analysis display
  • Analysis results now display in sorted order from largest to smallest match percentage
  • Application now supports uploading instructor supplied (base) files that can be included in analysis to narrow down meaningful results
  • Updated API to include record types on polymorphic objects
  • Revamped API calls so there should be no more 500 errors. Invalid data will be returned as empty objects with the same type as the method signature specifies. Invalid identifiers in returned data are now all set to -1 by default
10/10/10:
  • database schema migrated so that assignment names and file names are now 1024 characters max length, rather than 40
  • moss result pages (the highlighted code) is displayed dynamically from the database rather than from static files
  • first pass API implemented...right now it is data access only, there are no features to modify data in the db
  • any new reports that are run no longer are anonymized. this is because we do analysis in house, so there is no ferpa concerns
  • custom error handling - rather than getting default 404, 403, and 500 errors from apache, they are now styled using the same templates that comoto uses to render the rest of the app
  • new about page replacing the old footer
  • place holder page put in for API documentation
